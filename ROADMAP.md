# Pulsing Roadmap

本文档记录 Pulsing 项目的发展路线和规划。

## 当前状态 (v0.1.x)

- ✅ Actor System 核心实现
- ✅ Gossip 协议集群发现 (SWIM)
- ✅ HTTP/2 传输层
- ✅ Python 绑定 (PyO3)
- ✅ 流式消息支持
- ✅ AutoGen 集成
- ✅ LangGraph 集成
- ✅ 分布式队列

---

## 短期计划 (v0.2.x)

### Agent 框架集成增强

- [ ] **Dify 集成** - 作为 Dify 的分布式工作流运行时
  - [ ] Dify 节点 Actor 化 - 将工作流节点映射为 Pulsing Actor
  - [ ] 节点级分布式执行 - 单个工作流的节点可跨机器并行
  - [ ] LLM 节点卸载 - 将 LLM 推理节点调度到 GPU 服务器
  - [ ] 流式输出支持 - 优化 SSE 长连接通信
  - [ ] POC 验证 - 性能对比测试

### 核心功能

- [ ] Actor 监督树 (Supervision)
- [ ] 更多序列化格式支持
- [ ] 性能优化和基准测试

### 高性能传输

- [ ] **共享内存支持** - 同节点 Actor 间零拷贝通信
  - [ ] 本地 Actor 间共享内存传输 - 避免序列化开销
  - [ ] 大型数据零拷贝 - Tensor/ndarray 直接共享
  - [ ] 内存池管理 - 高效的共享内存分配与回收
  - [ ] 自动降级 - 跨节点时自动切换到网络传输

---

## 中期计划 (v0.3.x)

### Dify 深度集成

- [ ] Dify Plugin 模式 - 作为可选的分布式运行时插件
- [ ] 工作流状态分布式存储 - 减少数据库压力
- [ ] 故障恢复和重试机制
- [ ] 与 Dify 社区合作推进

### 高性能传输进阶

- [ ] **RDMA 支持** - 跨节点高性能通信
  - [ ] GPU Direct RDMA - GPU 间直接数据传输
  - [ ] InfiniBand 支持 - HPC 场景高带宽通信
- [ ] **Plasma Store 集成** - 跨进程大对象共享

### 基础设施

- [ ] Kubernetes Operator
- [ ] 可观测性增强 (Metrics/Tracing)
- [ ] 持久化存储后端

---

## 长期愿景

### 成为 LLM 应用的通用分布式运行时

```
┌─────────────────────────────────────────────────────────────┐
│                    Pulsing 生态系统                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   │
│   │ AutoGen │   │LangGraph│   │  Dify   │   │  其他   │    │
│   └────┬────┘   └────┬────┘   └────┬────┘   └────┬────┘   │
│        │             │             │             │         │
│        └─────────────┴──────┬──────┴─────────────┘         │
│                             │                               │
│                    ┌────────▼────────┐                     │
│                    │  Pulsing Core   │                     │
│                    │  Actor System   │                     │
│                    └────────┬────────┘                     │
│                             │                               │
│        ┌────────────────────┼────────────────────┐         │
│        │                    │                    │         │
│   ┌────▼────┐         ┌─────▼─────┐        ┌────▼────┐    │
│   │  GPU    │         │   CPU     │        │  Edge   │    │
│   │ Cluster │         │  Cluster  │        │ Devices │    │
│   └─────────┘         └───────────┘        └─────────┘    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 目标

1. **零依赖部署** - 保持轻量级，无需额外基础设施
2. **框架无关** - 支持主流 LLM 应用框架
3. **云原生** - 与 Kubernetes 深度集成
4. **高性能** - 满足生产级 LLM 推理需求
5. **零拷贝通信** - 共享内存 + RDMA 实现最小数据传输开销
