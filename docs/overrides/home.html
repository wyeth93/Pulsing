What is Pulsing?

Pulsing is a lightweight distributed Actor framework designed for building scalable LLM inference services. It provides location-transparent messaging, automatic cluster discovery using the SWIM protocol, high-performance async processing with Tokio, and seamless Python integration via PyO3. Build distributed AI systems with minimal boilerplate and maximum performance.

Learn More

Location Transparent Messaging

Send messages to actors without knowing their physical location. Pulsing automatically routes messages across the cluster using efficient HTTP/2 transport.

Actors can be local or remote - your code stays the same. Scale from single-node to distributed deployment without changing a line of application logic.

Learn More 
@as_actor
class Worker:
    async def process(self, data):
        return await self.model.generate(data)

# Send to any worker - local or remote
result = await worker.process(input_data)

Automatic Cluster Discovery

Built-in SWIM protocol enables automatic node discovery and failure detection. No external coordination service required - nodes find each other and form clusters automatically.

Gossip-based protocol ensures consistency across the cluster with minimal network overhead. Nodes join and leave gracefully with automatic state reconciliation.

Learn More 
# Start a node - it auto-discovers peers
system = ActorSystem(
    bind_addr="0.0.0.0:8080",
    seeds=["node1:8080", "node2:8080"]
)

# New nodes automatically join the cluster
await system.start()

High Performance Async Processing

Built on Tokio's high-performance async runtime, Pulsing handles thousands of concurrent actors with minimal overhead. Each actor runs in its own lightweight task.

Perfect for LLM inference where you need to manage many concurrent requests efficiently. Native Rust core with Python bindings delivers both safety and speed.

Learn More 
# Spawn thousands of actors
workers = [
    system.spawn(Worker, name=f"worker-{i}")
    for i in range(1000)
]

# All run concurrently with minimal overhead
results = await asyncio.gather(*[
    w.process(data) for w, data in zip(workers, inputs)
])

Python-First Design

Write actors in Python with the @as_actor decorator. No boilerplate, no complex setup - just define your class and it becomes a distributed actor.

Seamless integration with Python ML frameworks like PyTorch, Transformers, and vLLM. Build production-ready inference services in pure Python.

Learn More 
from pulsing import as_actor, ActorSystem

@as_actor
class LLMService:
    def __init__(self, model_name: str):
        self.model = AutoModelForCausalLM.from_pretrained(model_name)
    
    async def generate(self, prompt: str) -> str:
        return self.model.generate(prompt)
